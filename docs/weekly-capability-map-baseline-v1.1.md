# Weekly Capability Map Baseline v1.1

## Purpose

This is the first weekly-refresh baseline for Kevin's accumulated work.

It turns prior public-safe artifacts into a planning surface with:

- Capability nodes.
- Work lines.
- Needs.
- Current specs.
- Current artifacts.
- Future outlook.
- Kevin / partner / Codex / automation split.

This baseline is public-safe. It does not include real candidate data, private mail, account identifiers, compensation amounts, local paths, or restricted workflow records.

## Baseline Summary

Kevin's current work is best understood as a social-welfare technology operating system rather than a single job description.

The main pattern is:

1. Find a real service or planning need.
2. Turn it into a tool, content system, report, schema, workflow, or workboard.
3. Use Codex to draft, structure, verify, and publish safe artifacts.
4. Keep human approval for sensitive decisions.
5. Gradually identify which parts can move to partners or automation.

## Work Lines

The work lines are stored in `weekly/capability-map-work-lines-v1.1.csv`.

They can be imported into a spreadsheet or Notion database and reviewed weekly.

## Weekly Refresh Rule

Each weekly refresh should answer:

1. What changed this week?
2. Which capability node did it strengthen?
3. Which need does it answer?
4. What is the current spec or artifact?
5. What is the future outlook?
6. Which parts are Kevin / partner / Codex / automation?
7. What should remain private?
8. What is the smallest next step?

## Public Boundary

Public weekly updates may include:

- High-level work-line descriptions.
- Public artifact links.
- Fictional or de-identified examples.
- Public-safe plans, templates, and schemas.
- No-op statements when nothing changed.

Public weekly updates must not include:

- Real candidate records.
- Private messages.
- Phone, email, account identifiers, tokens, or local paths.
- Compensation amounts or private HR details.
- Any source that Kevin has not approved for public use.
