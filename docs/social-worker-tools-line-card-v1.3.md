# 社工可用工具開發 Line Card v1.3

## 目的

把「社工可用工具開發」從能力地圖上的方向，拆成可檢核、可交辦、可試作的小工作線。

這張卡的重點不是建立真正的個案系統，而是先把公開安全的工具需求整理清楚，讓 Kevin 可以判斷未來哪些工作要自己做、哪些可以交給夥伴、哪些可以讓 Codex 打底、哪些只適合自動化提醒。

## 需求

Kevin 目前需要的是一個能協助社工相關工作的工具規劃層：

- 快速整理公開資源與更新狀態。
- 把需求盤點轉成 checklist，而不是散落在對話裡。
- 把會談前準備、會後追蹤與 proof 欄位做成模板。
- 用小型 trial task 測試夥伴是否能協助資料整理、清單化、初稿與 QA。
- 讓沒有新資料的一週也能用 no-op checkpoint 正確收尾。

## 目前規格

| 模組 | 現在可做 | 不做什麼 |
| --- | --- | --- |
| 公開資源查找與新鮮度卡 | 整理公開來源、更新日期、適用對象與缺欄 | 不接私人個案資料 |
| 需求盤點 checklist | 把公開安全情境拆成問題清單與人工覆核欄 | 不做資格、診斷、法律或醫療判定 |
| 會談與服務紀錄準備包 | 建立空白或虛構模板、追蹤欄與 proof 欄 | 不公開真實會談內容 |
| 工具試作與回饋迴圈 | 用 public-safe trial task 測試夥伴協作 | 不主動寄送試作題給真人 |

## 分工

| 工作 | Kevin 做 | 夥伴做 | Codex 做 | 自動化 |
| --- | --- | --- | --- | --- |
| 資源查找 | 定義分類與優先順序 | 蒐集公開來源與更新日期 | 正規化欄位與產出摘要 | 連結與新鮮度檢查 |
| 需求盤點 | 定義判斷邊界 | 轉成問題清單初稿 | 補欄位、stop condition、測試案例 | 缺欄提醒 |
| 服務紀錄模板 | 定義必要欄位 | 整理模板初稿 | 轉成 Markdown / CSV / JSON | no-op reminder |
| 試作回饋 | 判斷題目是否貼近真實需要 | 完成 public-safe 小任務 | 產生 rubric 與 feedback loop | 提醒回饋期限 |

## 工作檢核點

| 檢核點 | 通過標準 |
| --- | --- |
| 公開安全 | 只使用公開、空白、虛構或去識別化內容 |
| 可交辦 | 任務能在一頁 brief 內說清楚輸入、輸出、proof |
| 可覆核 | 每個輸出都有 Kevin 或指定 reviewer 欄位 |
| 可停止 | 高風險、不清楚或涉及私人資料時停止 |
| 可週更 | 有 changed / no-op / blocked 三種結果 |

## 未來展望

v1.4 可以挑一個模組做最小可行 prototype brief。建議先從「公開資源查找與新鮮度卡」開始，因為它最容易交給夥伴，也最適合 Codex 協助轉成表格、卡片與驗證流程。

若未來要進入 private pilot，必須另開 private implementation 邊界，先處理資料政策、人工覆核、權限、保存與刪除規則。noindex 不能當作隱私保護。
