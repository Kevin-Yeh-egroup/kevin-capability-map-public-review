# Weekly Capability Dashboard v1.2

## Purpose

v1.2 turns the weekly capability map into a decision dashboard.

The goal is to let Kevin quickly see:

- What changed.
- What is newly available as an artifact.
- What can be delegated to a partner.
- What Codex can handle.
- What could become automation.
- What is blocked or private-only.
- What the next week should focus on.

This is a public-safe dashboard. It does not include private source records, real candidates, private messages, compensation amounts, account identifiers, tokens, or local paths.

## Dashboard Sections

| Section | Question |
| --- | --- |
| Status Metrics | How many capability nodes, work lines, delegation candidates, and blockers exist? |
| Changed Work Lines | What changed since the previous baseline? |
| Delegation Snapshot | What should Kevin keep, what can partners do, what can Codex do, what might automation do? |
| Blocked / Private Only | What must not move into the public artifact? |
| Next Week Candidates | Which 3-5 actions are most useful next? |
| Checkpoint Card | Is the run proceed, no-op, or blocked? |

## Practical Reading

The strongest signal is that Kevin's work is now structured enough to support weekly review.

The dashboard should not try to replace Kevin's judgment. It should reduce the time needed to see what changed, where the work belongs, and what can be safely delegated.

## Public Boundary

Public dashboard updates may include:

- Public artifact names.
- High-level workflow descriptions.
- Fictional or de-identified examples.
- Public-safe delegation categories.
- No-op checkpoints.

Public dashboard updates must not include:

- Real candidate records.
- Private messages or inbox content.
- Compensation amounts.
- Local paths.
- Account identifiers or tokens.
- Internal workforce-war-room records.
