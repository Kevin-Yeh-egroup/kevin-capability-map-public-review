# Content Research Trial Task v1.0

## Purpose

Test whether a content / research partner can turn scattered source notes into a public-safe, dated, reviewable work packet that reduces Kevin's planning load.

This is a fictional public-review template. It is not sent to a real person from this package.

## Role Tested

`content_research`

## Task Scope

Create one research packet from fictional source cards:

1. Build a source table with title, source type, date checked, key claim, confidence, and missing fields.
2. Write a short public-safe brief for a non-specialist reader.
3. Convert the brief into a checklist Kevin can review.
4. Mark anything that needs Kevin approval before use.
5. Write a short revision note after feedback.

## Expected Proof

| Proof | Pass Signal |
| --- | --- |
| Source table | Dates, missing fields, and confidence are explicit. |
| Public-safe brief | No private source text, no invented facts, no sensitive details. |
| Checklist | Items are concrete enough to assign or verify. |
| Redaction notes | Candidate names stop conditions and approval gates. |
| Revision note | Feedback is applied and summarized. |

## Stop Conditions

Stop and ask Kevin before continuing if the task touches:

- Real person data.
- Private accounts or inboxes.
- Compensation, offer, or employment decision topics.
- External publishing.
- Production data writes.

## Review Rubric

| Dimension | What To Check |
| --- | --- |
| D1 work-line understanding | Connects research output to Kevin's planning and tool-building workflows. |
| D2 execution reliability | Finishes the exact requested packet with done / blocked / missing fields. |
| D3 evidence quality | Distinguishes source facts, interpretation, and unknowns. |
| D4 communication | Gives Kevin a short decision-ready summary. |
| D5 trainability | Applies feedback without broad drift. |
| D6 risk gate awareness | Stops before sensitive or external actions. |
| D7 Codex / tool collaboration | Can turn the packet into prompts, tables, or checklists. |
| D8 role depth | Shows useful research judgment and plain-language translation. |

## Pass Definition

The trial passes only as a planning signal when the packet is reviewable, source-aware, short enough for Kevin to use, and safe by default. It does not create hiring, rejection, or compensation authority.
